# Linux 进程管理工具：评分要求对照与验收

## 结论

本项目完整覆盖题目 13 的五项要求，并通过网络、磁盘、进程详情、进程树、PID 复用保护和真实 `fork()` 演示体现功能丰富度与创新性。最终成绩仍由实践报告、系统实现和答辩共同决定，本表用于提供可核验的程序证据。

## 题目要求映射

| 要求 | 实现证据 | 演示方式 | 验证方式 |
|---|---|---|---|
| CPU/内存/进程实时显示 | 手写解析 `/proc/stat`、`/proc/meminfo`、`/proc/[pid]/stat` | 观察 CPU、每核、内存与进程表每秒更新 | `go test ./... -race` |
| 创建、杀死、优先级调整 | `exec.Command`、`kill(2)`、`setpriority(2)`；PID+starttime 防误操作 | 启动 `sleep 20`，调整 nice，发送 TERM | 界面事件轨和进程消失 |
| 图形界面与历史曲线 | Wails 的 Linux GTK/WebKitGTK 后端；ECharts SVG 曲线 | 展示 CPU、内存、网络、磁盘 60 点历史 | `npm run build`、Wails 生产构建 |
| `/proc` 文件解析 | CPU、内存、负载、进程、网络、磁盘均为手写解析 | 答辩展示字段映射和解析测试 | `internal/procfs/*_test.go` |
| `fork()`、`kill()` 系统调用 | cgo 调用 libc `fork()`，子进程立即 `execv`，父进程 `wait4`；kill 使用 `syscall.Kill` | “new → 运行 fork”，观察 sleep 子进程和事件轨 | `TestForkSleep` |

## 优秀档增量

- 网络：`/proc/net/dev` 累计字节差分为上下行 bytes/s。
- 磁盘：`/proc/diskstats` 的 512-byte sector 差分为读写 bytes/s，`statfs(2)` 获取根分区容量。
- 进程：平铺/树视图、用户、搜索、排序、按需详情、命令行、I/O、FD、上下文切换。
- 安全：拒绝 PID 0/1/自身，限制信号，使用 PID+starttime 防 PID 复用。
- 工程性：race test、vet、严格 TypeScript、可选采集独立降级、无重叠轮询。
